빛 반응 책상등

이번 시간의 출발점은 작은 장치가 바로 반응하는 경험입니다. 빛 반응 책상등 활동은 학생이 처음부터 어려운 문법을 외우기보다, 내가 바꾼 코드 한 줄이 눈앞의 색·소리·반응을 바꾸는 장면을 먼저 만나도록 설계했습니다.

이 단원의 진짜 가치는 결과물 하나를 완성하는 데 있지 않습니다. 중요한 것은 코드를 화면 속 글자가 아니라 현실을 움직이는 명령으로 느끼는 것입니다. 학생은 숫자, 조건, 입력, 출력이 추상 개념이 아니라 책상 위 장치의 행동으로 나타난다는 것을 경험합니다.

처음에는 코드 전체를 완벽히 이해하지 않아도 괜찮습니다. 대신 어떤 값을 바꾸었고, 그 결과 장치가 어떻게 달라졌는지를 말할 수 있어야 합니다. 이 작은 설명 능력이 이후 데이터 분석과 AI 코드 검증의 기초가 됩니다.

시각자료

이미지 선택센서 흐름 다이어그램

조도 센서값과 기준값 비교가 출력 색을 바꾸는 장면을 설명합니다.

🧠 컴퓨팅 사고력 · 큰 미션을 작은 조각으로 나누기

한 번에 완성하려고 하면 어디서 문제가 생겼는지 알기 어렵습니다. 오늘은 가장 작은 기능을 먼저 만들고, 눈으로 확인하고, 그 다음 기능을 붙입니다. 이 짧은 순환이 안전한 코딩과 바이브코딩의 기본입니다.

단계별 활동

단계 1 — 완성 장면 먼저 보기

교사는 완성된 장치를 30초 정도만 보여줍니다. 긴 설명보다 “어? 코드가 진짜 움직이네?”라는 첫 반응이 중요합니다.

단계 2 — 가장 작은 코드 실행하기

전체 기능을 한 번에 만들지 않습니다. LED 하나 켜기, 버튼 하나 읽기처럼 가장 작은 성공부터 확인합니다.

단계 3 — 한 곳만 바꿔보기

색상값, 대기 시간, 기준값, 메시지 중 한 곳만 바꿉니다. 바꾼 곳과 달라진 결과를 짝지어 기록합니다.

단계 4 — 팀 규칙 만들기

우리 팀만의 색, 감정, 기준, 게임 규칙을 정합니다. 이때 코드는 자기표현의 도구가 됩니다.

단계 5 — 친구에게 설명하기

“이 줄을 바꾸면 이렇게 달라진다”를 한 문장으로 말합니다. 설명하지 못하면 다시 코드로 돌아갑니다.

프롬프트 설계

AI를 사용할 때는 바로 “만들어줘”라고 묻지 않습니다. 아래 요소를 채워 넣으면 AI 답변이 훨씬 안정적입니다.

맥락Pico 2 W, Grove Shield, MicroPython, 팀 활동
목표조도 센서값에 따라 LED 색이 바뀌는 책상등을 만든다.
제약외부 라이브러리 최소화, 한 번에 한 기능만 추가, 실행 후 검증
예시 결과빛 반응 책상등
재사용이전 차시의 코드/데이터/기준값을 재사용

🤖 샘플 프롬프트

Raspberry Pi Pico 2 W와 Grove Shield를 사용하는 고등학생 수업입니다. 오늘 목표는 '빛 반응 책상등'입니다. 코드는 너무 길지 않게 만들고, 학생이 바꿔볼 기준값이나 메시지를 표시해 주세요. 각 줄에 한국어 주석을 달고, 실행 후 확인할 체크리스트도 함께 작성해 주세요.

받을 수 있는 샘플 코드/데이터

from time import sleep

LOW = 300
HIGH = 700

while True:
    value = read_sensor()  # 조도/소리/온습도 센서값
    print("sensor:", value)

    if value < LOW:
        state = "낮음"
    elif value < HIGH:
        state = "보통"
    else:
        state = "높음"

    print("state:", state)
    sleep(1)

# 우리 팀 데이터에 맞게 LOW와 HIGH를 바꿔보세요.

이 코드는 출발점입니다. 장비, 센서 포트, 라이브러리 이름은 학교 키트에 맞게 조정할 수 있습니다. 중요한 것은 구조를 읽고, 바꾼 부분을 설명하고, 실행 결과로 검증하는 것입니다.

검증 체크리스트

  • 오늘 만들 것이 실제로 작동하거나 화면에 표시되나요?
  • 내가 바꾼 코드/데이터/기준값을 한 문장으로 설명할 수 있나요?
  • 친구 팀과 결과가 다르다면 그 이유를 하나 이상 말할 수 있나요?
  • AI를 사용했다면 질문, 답변, 수정, 검증 방법을 기록했나요?
  • 결론을 말할 때 데이터의 한계도 함께 말했나요?

다음 단계로 가는 이유

우리 팀이 정한 밝다의 기준은 모든 자리에서 맞을까?

이 질문은 다음 차시의 출발점입니다. 오늘 만든 결과가 충분히 재미있었다면, 이제 그것을 더 오래 기록하거나, 더 보기 좋게 표현하거나, 친구들과 함께 합쳐 볼 차례입니다.

수정·확장 미션

  • 기본 결과에 우리 팀만의 색상, 메시지, 기준값을 추가합니다.
  • 측정 조건이나 사용자의 행동이 달라지면 결과가 어떻게 바뀌는지 확인합니다.
  • 가능하다면 사진, 표, 그래프, 링크 중 하나로 결과를 남깁니다.

연습문제

  1. 빛 반응 책상등 활동에서 기준값이나 메시지 하나를 바꾸고, 바꾸기 전후 결과를 비교해 보세요.
  2. 오늘 데이터나 코드에서 친구 팀과 달랐던 점을 하나 찾고, 왜 달랐는지 가능한 이유를 적어 보세요.
  3. AI에게 오늘 활동을 도와달라고 묻는 좋은 프롬프트를 작성해 보세요. 목표, 데이터 구조, 제약 조건을 반드시 포함하세요.
  4. 도전: 오늘 결과물을 다음 차시 프로젝트와 연결하려면 어떤 열, 그래프, 화면 요소가 추가되어야 할지 설계해 보세요.

마무리

오늘의 목표는 완벽한 결과물을 한 번에 만드는 것이 아니라, 작은 기능을 만들고 검증하고 다음 단계로 확장하는 경험을 쌓는 것입니다. 이 패턴이 반복되면 피코 장치, 데이터 분석, AI 협업, 웹 대시보드가 따로 떨어진 기술이 아니라 하나의 프로젝트 흐름으로 연결됩니다.
