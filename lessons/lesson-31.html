31차시 · 메이커 페어 프로젝트

데이터 수집과 업로드

이번 시간은 팀 프로젝트를 전시 가능한 작품으로 완성하는 경험입니다. 지금까지의 모든 조각, 즉 피코 장치·센서 데이터·AI 활용 기록·대시보드·GitHub 문서가 하나의 프로젝트로 모입니다.

핵심은 재미있는 장치, 설득력 있는 데이터, 책임 있는 AI 기록을 하나로 묶는 것입니다. 작품은 재미있어야 하지만 장난감으로 끝나면 안 되고, 데이터는 설득력 있어야 하지만 지루한 보고서로 끝나면 안 됩니다.

좋은 최종 프로젝트는 친구가 직접 만져보고, 화면으로 데이터를 확인하고, 발표를 들은 뒤 ‘다음 버전은 이렇게 더 좋아질 수 있겠다’고 말할 수 있어야 합니다.

시각자료

이미지 선택현장 수집 다이어그램

측정 맥락 메모와 업로드가 함께 있어야 데이터가 설명력을 갖는다는 점을 보여줍니다.

🧠 컴퓨팅 사고력 · 큰 미션을 작은 조각으로 나누기

한 번에 완성하려고 하면 어디서 문제가 생겼는지 알기 어렵습니다. 오늘은 가장 작은 기능을 먼저 만들고, 눈으로 확인하고, 그 다음 기능을 붙입니다. 이 짧은 순환이 안전한 코딩과 바이브코딩의 기본입니다.

단계별 활동

단계 1 — 최소 완성 기준 정하기

장치, 데이터, 대시보드, 기록, 발표 중 오늘 반드시 완성할 최소 기준을 정합니다.

단계 2 — 팀 역할 나누기

메이커, 코더, 데이터 탐정, 디자이너/발표자로 나누고 부족한 부분을 보완합니다.

단계 3 — 피드백 받고 고치기

친구가 직접 체험하고 이해하기 어려운 부분을 표시하게 합니다.

단계 4 — AI 기록 정리하기

AI를 쓴 프롬프트와 수정·검증 내용을 발표 자료와 GitHub에 남깁니다.

단계 5 — 다음 버전 제안하기

완성했다고 끝내지 않고, 한계와 다음 개선 기능을 제안합니다.

프롬프트 설계

AI를 사용할 때는 바로 “만들어줘”라고 묻지 않습니다. 아래 요소를 채워 넣으면 AI 답변이 훨씬 안정적입니다.

맥락Python, CSV, Streamlit, 웹 GitHub, AI 도구
목표실제 데이터를 수집하고 클라우드에 올린다.
제약웹 UI 중심, CLI 사용하지 않기, 템플릿 복사 후 수정
예시 결과프로젝트 데이터셋
재사용이전 차시의 코드/데이터/기준값을 재사용

🤖 샘플 프롬프트

우리 팀 프로젝트는 '프로젝트 데이터셋'입니다. 발표 자료를 문제-장치-데이터-대시보드-AI 활용-한계-다음 버전 순서로 구성해 주세요. 과장하지 말고 데이터로 말하게 해 주세요.

받을 수 있는 샘플 코드/데이터

from time import sleep

LOW = 300
HIGH = 700

while True:
    value = read_sensor()  # 조도/소리/온습도 센서값
    print("sensor:", value)

    if value < LOW:
        state = "낮음"
    elif value < HIGH:
        state = "보통"
    else:
        state = "높음"

    print("state:", state)
    sleep(1)

# 우리 팀 데이터에 맞게 LOW와 HIGH를 바꿔보세요.

이 코드는 출발점입니다. 장비, 센서 포트, 라이브러리 이름은 학교 키트에 맞게 조정할 수 있습니다. 중요한 것은 구조를 읽고, 바꾼 부분을 설명하고, 실행 결과로 검증하는 것입니다.

검증 체크리스트

  • 오늘 만들 것이 실제로 작동하거나 화면에 표시되나요?
  • 내가 바꾼 코드/데이터/기준값을 한 문장으로 설명할 수 있나요?
  • 친구 팀과 결과가 다르다면 그 이유를 하나 이상 말할 수 있나요?
  • AI를 사용했다면 질문, 답변, 수정, 검증 방법을 기록했나요?
  • 결론을 말할 때 데이터의 한계도 함께 말했나요?

다음 단계로 가는 이유

데이터 맥락은 충분히 기록됐을까?

이 질문은 다음 차시의 출발점입니다. 오늘 만든 결과가 충분히 재미있었다면, 이제 그것을 더 오래 기록하거나, 더 보기 좋게 표현하거나, 친구들과 함께 합쳐 볼 차례입니다.

수정·확장 미션

  • 기본 결과에 우리 팀만의 색상, 메시지, 기준값을 추가합니다.
  • 측정 조건이나 사용자의 행동이 달라지면 결과가 어떻게 바뀌는지 확인합니다.
  • 가능하다면 사진, 표, 그래프, 링크 중 하나로 결과를 남깁니다.

연습문제

  1. 데이터 수집과 업로드 활동에서 기준값이나 메시지 하나를 바꾸고, 바꾸기 전후 결과를 비교해 보세요.
  2. 오늘 데이터나 코드에서 친구 팀과 달랐던 점을 하나 찾고, 왜 달랐는지 가능한 이유를 적어 보세요.
  3. AI에게 오늘 활동을 도와달라고 묻는 좋은 프롬프트를 작성해 보세요. 목표, 데이터 구조, 제약 조건을 반드시 포함하세요.
  4. 도전: 오늘 결과물을 다음 차시 프로젝트와 연결하려면 어떤 열, 그래프, 화면 요소가 추가되어야 할지 설계해 보세요.

마무리

오늘의 목표는 완벽한 결과물을 한 번에 만드는 것이 아니라, 작은 기능을 만들고 검증하고 다음 단계로 확장하는 경험을 쌓는 것입니다. 이 패턴이 반복되면 피코 장치, 데이터 분석, AI 협업, 웹 대시보드가 따로 떨어진 기술이 아니라 하나의 프로젝트 흐름으로 연결됩니다.
