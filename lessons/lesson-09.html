09차시 · 데이터 탐구

우리 반 데이터 산책

이번 시간은 센서값이 데이터가 되는 경험입니다. 앞에서 만든 장치는 이제 단순한 장난감이 아니라 교실과 학교 공간을 관찰하는 작은 측정 도구가 됩니다.

핵심은 한 번의 값보다 반복 측정과 맥락 기록이 더 강한 근거가 된다는 것입니다. 센서값 하나는 순간의 스냅샷일 뿐이지만, 시간·장소·조건과 함께 쌓이면 질문에 답하는 데이터가 됩니다.

데이터 수업은 표와 그래프를 그리는 기술만 배우는 시간이 아닙니다. 좋은 질문을 만들고, 측정 조건을 기록하고, 이상한 값을 무시하지 않고 이유를 찾는 과정 전체가 데이터 리터러시입니다.

이번 시간의 미션

팀별로 다른 위치를 측정해 공동 데이터셋을 만든다.

  • 오늘 만들 것: 교실 장소별 데이터
  • 핵심 개념: 협력 수집 · 장소 차이
  • 저장할 산출물: 장소별 측정 데이터
측정 질문→공통 표→그래프→한계

🧠 컴퓨팅 사고력 · 큰 미션을 작은 조각으로 나누기

한 번에 완성하려고 하면 어디서 문제가 생겼는지 알기 어렵습니다. 오늘은 가장 작은 기능을 먼저 만들고, 눈으로 확인하고, 그 다음 기능을 붙입니다. 이 짧은 순환이 안전한 코딩과 바이브코딩의 기본입니다.

단계별 활동

단계 1 — 측정 질문 정하기

그냥 값을 모으지 않습니다. “우리 반은 언제 가장 시끄러울까?”처럼 답하고 싶은 질문을 먼저 정합니다.

단계 2 — 공통 양식으로 기록하기

team, timestamp, place, sensor value, note처럼 모두가 같은 열 이름을 사용합니다. 데이터는 형식이 맞아야 합칠 수 있습니다.

단계 3 — 이상한 값 남겨두기

갑자기 튄 값은 바로 지우지 않습니다. 박수, 창문, 사람 이동 같은 맥락을 note에 남깁니다.

단계 4 — 그래프로 패턴 보기

표에서는 잘 보이지 않던 시간 변화, 장소 차이, 팀별 차이를 그래프로 확인합니다.

단계 5 — 결론과 한계 나누기

데이터가 말해주는 것과 아직 말할 수 없는 것을 구분합니다. 좋은 분석은 한계를 숨기지 않습니다.

프롬프트 설계

AI를 사용할 때는 바로 “만들어줘”라고 묻지 않습니다. 아래 요소를 채워 넣으면 AI 답변이 훨씬 안정적입니다.

맥락Pico 2 W, Grove Shield, MicroPython, 팀 활동
목표팀별로 다른 위치를 측정해 공동 데이터셋을 만든다.
제약외부 라이브러리 최소화, 한 번에 한 기능만 추가, 실행 후 검증
예시 결과교실 장소별 데이터
재사용이전 차시의 코드/데이터/기준값을 재사용

🤖 샘플 프롬프트

Raspberry Pi Pico 2 W와 Grove Shield를 사용하는 고등학생 수업입니다. 오늘 목표는 '교실 장소별 데이터'입니다. 코드는 너무 길지 않게 만들고, 학생이 바꿔볼 기준값이나 메시지를 표시해 주세요. 각 줄에 한국어 주석을 달고, 실행 후 확인할 체크리스트도 함께 작성해 주세요.

받을 수 있는 샘플 코드/데이터

from time import sleep

LOW = 300
HIGH = 700

while True:
    value = read_sensor()  # 조도/소리/온습도 센서값
    print("sensor:", value)

    if value < LOW:
        state = "낮음"
    elif value < HIGH:
        state = "보통"
    else:
        state = "높음"

    print("state:", state)
    sleep(1)

# 우리 팀 데이터에 맞게 LOW와 HIGH를 바꿔보세요.

이 코드는 출발점입니다. 장비, 센서 포트, 라이브러리 이름은 학교 키트에 맞게 조정할 수 있습니다. 중요한 것은 구조를 읽고, 바꾼 부분을 설명하고, 실행 결과로 검증하는 것입니다.

검증 체크리스트

  • 오늘 만들 것이 실제로 작동하거나 화면에 표시되나요?
  • 내가 바꾼 코드/데이터/기준값을 한 문장으로 설명할 수 있나요?
  • 친구 팀과 결과가 다르다면 그 이유를 하나 이상 말할 수 있나요?
  • AI를 사용했다면 질문, 답변, 수정, 검증 방법을 기록했나요?
  • 결론을 말할 때 데이터의 한계도 함께 말했나요?

다음 단계로 가는 이유

여러 팀이 나눠 측정하면 무엇이 좋아질까?

이 질문은 다음 차시의 출발점입니다. 오늘 만든 결과가 충분히 재미있었다면, 이제 그것을 더 오래 기록하거나, 더 보기 좋게 표현하거나, 친구들과 함께 합쳐 볼 차례입니다.

수정·확장 미션

  • 기본 결과에 우리 팀만의 색상, 메시지, 기준값을 추가합니다.
  • 측정 조건이나 사용자의 행동이 달라지면 결과가 어떻게 바뀌는지 확인합니다.
  • 가능하다면 사진, 표, 그래프, 링크 중 하나로 결과를 남깁니다.

연습문제

  1. 우리 반 데이터 산책 활동에서 기준값이나 메시지 하나를 바꾸고, 바꾸기 전후 결과를 비교해 보세요.
  2. 오늘 데이터나 코드에서 친구 팀과 달랐던 점을 하나 찾고, 왜 달랐는지 가능한 이유를 적어 보세요.
  3. AI에게 오늘 활동을 도와달라고 묻는 좋은 프롬프트를 작성해 보세요. 목표, 데이터 구조, 제약 조건을 반드시 포함하세요.
  4. 도전: 오늘 결과물을 다음 차시 프로젝트와 연결하려면 어떤 열, 그래프, 화면 요소가 추가되어야 할지 설계해 보세요.

마무리

오늘의 목표는 완벽한 결과물을 한 번에 만드는 것이 아니라, 작은 기능을 만들고 검증하고 다음 단계로 확장하는 경험을 쌓는 것입니다. 이 패턴이 반복되면 피코 장치, 데이터 분석, AI 협업, 웹 대시보드가 따로 떨어진 기술이 아니라 하나의 프로젝트 흐름으로 연결됩니다.
