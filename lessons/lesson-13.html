13차시 · 데이터 탐구

중간 미니 발표

오늘은 이런 일을 해요

이제 Grove 센서는 단순히 반응하는 부품이 아니라 데이터를 만드는 관찰 도구가 됩니다. 중간 미니 발표에서는 한 번 본 값이 아니라 시간·장소·조건과 함께 쌓이는 값을 다룹니다.

좋은 데이터는 ‘숫자만 많은 표’가 아닙니다. 언제, 어디서, 어떤 조건에서 측정했는지 함께 적어야 나중에 그래프와 해석이 의미를 가집니다.

이 구간에서도 AI는 사용하지 않습니다. 먼저 우리 손으로 데이터를 만들고, 표를 고치고, 그래프를 보며 데이터 감각을 쌓습니다.

데이터 발표 자료근거 · 한계 · 피드백미니 발표, 피드백

연결 그림

그림 13-1정확한 연결 확인용

오늘은 발표 자료 정리만 기억하면 됩니다. 코드에서도 같은 포트 이름을 사용합니다.

따라하기

1. 측정 질문 정하기

그냥 값을 모으지 말고, 오늘 데이터로 답하고 싶은 질문을 먼저 적습니다.

2. 같은 형식으로 기록하기

team, timestamp, place, value, note처럼 같은 열 이름을 사용합니다.

3. 이상한 값 표시하기

튀는 값이나 빈 값은 바로 지우지 말고 표시합니다. 이유가 중요한 단서일 수 있습니다.

4. 그래프로 보기

표에서 잘 보이지 않는 시간 변화와 장소 차이를 그래프로 확인합니다.

5. 한계 말하기

데이터가 말해주는 것과 아직 말할 수 없는 것을 나눠서 정리합니다.

실습 코드

아래 내용은 오늘의 출발점입니다. 그대로 따라 한 뒤, 우리 팀 상황에 맞게 표시된 부분을 바꿔 보세요.

from time import sleep
from grove_helper import GroveLightSensor

sensor = GroveLightSensor(port="A0")
team = "A"
place = "window"

print("team,timestamp,place,light,note")

for i in range(5):
    value = sensor.read()
    timestamp = "09:%02d" % (i * 5)
    print(f"{team},{timestamp},{place},{value},")
    sleep(1)

# CSV로 옮길 때 열 이름과 순서를 그대로 지킵니다.

아직 AI는 사용하지 않아요

이번 단계에서는 직접 실행하고, 직접 관찰하고, 직접 기록하는 경험이 먼저입니다. AI는 14차시부터 안전하게 사용하는 법을 배운 뒤에 사용합니다.

  • 친구 팀과 결과가 다르면 그 이유를 말할 수 있나요?
