06차시 · 재미있는 손코딩

내 장치 설명하기

내가 만든 장치를 입력-처리-출력 구조로 설명한다.

오늘의 흐름

입력→조건/처리→색·소리·화면

오늘 만들 것

장치 설명 카드

왜 배우나요?

오늘 수업의 핵심은 '장치 설명 카드'이 실제로 눈앞에서 반응하는 경험입니다. 처음부터 모든 코드를 이해하려고 하기보다, 어느 줄을 바꾸면 결과가 달라지는지 관찰합니다.

팀원은 연결 담당, 코드 실행 담당, 기록 담당, 설명 담당으로 나눕니다. 역할은 다음 차시마다 바꿔서 모두가 장치와 코드에 손을 대게 합니다.

성공 기준은 화려한 작품이 아니라, 내가 바꾼 부분과 달라진 결과를 한 문장으로 설명하는 것입니다.

준비물

  • Raspberry Pi Pico 2 W + Grove Shield
  • 오늘 활동에 맞는 Grove 센서 또는 출력 모듈
  • USB 케이블, 노트북, 팀 기록지
  • CSV 양식 또는 활동 기록지

핵심 개념

입력 · 처리 · 출력 · 코드 읽기

따라하기

  1. 완성 예시를 30초 동안 먼저 보고 오늘 만들 결과를 상상합니다.
  2. 피코와 모듈을 연결하고 기본 코드가 실행되는지 확인합니다.
  3. 교재에서 표시한 숫자, 기준값, 메시지 중 하나만 먼저 바꿔봅니다.
  4. 결과가 달라지면 왜 달라졌는지 팀 기록지에 적습니다.
  5. 친구 팀과 결과를 비교하고 우리 팀만의 색/규칙/설명을 붙입니다.

실습 코드/데이터

# 반응속도 게임/장치 설명에 사용할 수 있는 시간 측정 구조입니다.
from time import ticks_ms, ticks_diff, sleep

start = ticks_ms()
sleep(1)
end = ticks_ms()
reaction_time = ticks_diff(end, start)
print("걸린 시간(ms):", reaction_time)

장비와 라이브러리에 따라 함수명이나 핀 번호는 달라질 수 있습니다. 중요한 것은 구조를 읽고 우리 팀 상황에 맞게 고치는 것입니다.

바꿔보기 미션

  • 기준값, 색상, 메시지, 그래프 제목 중 하나를 우리 팀 상황에 맞게 바꿉니다.
  • 바꾼 전/후 결과를 사진, 표, 문장 중 하나로 기록합니다.
  • 친구 팀과 결과가 다르면 데이터나 기준값이 어떻게 다른지 설명합니다.

생각 질문

작동은 되지만 설명 못 하는 코드는 이해한 코드일까?

저장할 산출물

설명 카드, 핵심 코드 주석

오류가 나면

  • 케이블 방향과 포트 번호를 먼저 확인합니다.
  • 데이터 열 이름이 예제 코드와 정확히 같은지 확인합니다.
  • 오류 메시지를 복사하기 전에 첫 줄과 마지막 줄을 읽습니다.
  • AI에게 물어볼 때는 코드, 오류 메시지, 내가 하려던 일을 함께 알려줍니다.
