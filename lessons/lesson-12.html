12차시 · 데이터 탐구

데이터로 규칙 만들기

센서값으로 쾌적·주의·나쁨 기준을 만든다.

오늘의 흐름

센서값→CSV 표→그래프→해석

오늘 만들 것

상태 분류 기준표

왜 배우나요?

이제 장치는 단순히 반응하는 물건이 아니라 데이터를 만드는 관찰 도구가 됩니다. 한 번 본 센서값이 아니라 시간, 장소, 조건과 함께 쌓인 값을 다룹니다.

팀별 데이터 형식이 다르면 나중에 합칠 수 없으므로 공통 열 이름과 단위를 지키는 것이 중요합니다.

결론을 빨리 내기보다, 데이터가 말해주는 것과 아직 말해주지 못하는 것을 구분합니다.

준비물

  • Raspberry Pi Pico 2 W + Grove Shield
  • 오늘 활동에 맞는 Grove 센서 또는 출력 모듈
  • USB 케이블, 노트북, 팀 기록지
  • CSV 양식 또는 활동 기록지

핵심 개념

분류 · 기준값 · 한계

따라하기

  1. 오늘의 측정 질문을 한 문장으로 정합니다.
  2. 시간, 장소, 조건, 센서값을 같은 표 형식으로 기록합니다.
  3. 값이 이상하거나 비어 있으면 바로 지우지 말고 이유를 메모합니다.
  4. 그래프나 기준표로 데이터가 보여주는 패턴을 찾습니다.
  5. 확실한 해석과 아직 모르는 점을 나눠서 발표 문장으로 정리합니다.

실습 코드/데이터

# 센서값을 읽고 기준에 따라 상태를 나누는 기본 구조입니다.
from time import sleep

LOW = 300
HIGH = 700

while True:
    value = read_sensor()  # 예: 조도, 소리, 온습도 센서값
    print("sensor:", value)

    if value < LOW:
        state = "낮음"
    elif value < HIGH:
        state = "보통"
    else:
        state = "높음"

    print("state:", state)
    sleep(1)

# 바꿔보기: LOW와 HIGH 기준값을 우리 팀 데이터에 맞게 바꿔보세요.

장비와 라이브러리에 따라 함수명이나 핀 번호는 달라질 수 있습니다. 중요한 것은 구조를 읽고 우리 팀 상황에 맞게 고치는 것입니다.

바꿔보기 미션

  • 기준값, 색상, 메시지, 그래프 제목 중 하나를 우리 팀 상황에 맞게 바꿉니다.
  • 바꾼 전/후 결과를 사진, 표, 문장 중 하나로 기록합니다.
  • 친구 팀과 결과가 다르면 데이터나 기준값이 어떻게 다른지 설명합니다.

저장할 산출물

오류가 나면

  • 케이블 방향과 포트 번호를 먼저 확인합니다.
  • 데이터 열 이름이 예제 코드와 정확히 같은지 확인합니다.
  • 오류 메시지를 복사하기 전에 첫 줄과 마지막 줄을 읽습니다.
  • AI에게 물어볼 때는 코드, 오류 메시지, 내가 하려던 일을 함께 알려줍니다.
