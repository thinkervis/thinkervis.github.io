챕터 5. 데이터 병아리반 — 숫자를 믿기 전에 질문하기

데이터 실습은 재미있지만 위험합니다. Copilot이 숫자를 말하면 그럴듯해 보이기 때문입니다. 데이터 수업의 핵심은 ‘분석해줘’가 아니라 ‘어떤 기준으로 분석했는지 설명해줘’입니다.

먼저 표를 이해했는지 확인합니다

수강생에게 CSV나 Excel을 주고 바로 인사이트를 달라고 하면 안 됩니다. 먼저 Copilot이 컬럼을 어떻게 이해했는지 확인해야 합니다. 컬럼 뜻을 잘못 이해하면 뒤의 분석도 모두 흔들립니다.

첨부한 CSV의 컬럼 구조를 먼저 설명해줘.
각 컬럼이 어떤 의미로 보이는지 추정하고, 분석 전에 확인해야 할 점을 알려줘.
그 다음 문의 유형 TOP 5, 평균 처리 시간이 긴 제품군, 재문의율이 높은 유형을 계산 기준과 함께 정리해줘.
잠깐만요 │ 계산 결과는 반드시 샘플로 검산합니다

상위 3개, 합계, 평균처럼 간단한 값은 사람이 직접 한두 개 확인해보는 습관을 들여야 합니다. Copilot이 계산 기준을 설명하지 못하면 결과를 그대로 믿으면 안 됩니다.

강사용 데이터 실습 운영법

좋은 데이터 실습은 작은 데이터로 시작합니다. 처음부터 복잡한 피벗이나 거대한 파일을 다루면 수강생은 Copilot보다 파일 처리에 지칩니다. 20~50행 정도의 샘플 데이터로 문의 유형, 처리 시간, 만족도처럼 이해하기 쉬운 컬럼을 사용하세요.

강사 “데이터 분석에서 첫 질문은 인사이트를 달라는 것이 아닙니다. 먼저 Copilot이 데이터를 어떻게 이해했는지 확인해야 합니다.”
수강생 예상 “결과가 맞는지 모르겠습니다.”
강사 답변 “좋은 의심입니다. 계산 기준을 설명하게 하고, 간단한 숫자는 직접 확인해봅시다. 데이터 분석에서 의심은 나쁜 태도가 아니라 좋은 태도입니다.”

Hermes판 더 친절한 강의 노트

이 장을 가르칠 때 강사는 먼저 수강생의 마음을 낮춰야 합니다. Copilot을 처음 배우는 사람은 “이걸 모르면 뒤처지는 것 아닐까”라는 불안과 “그래도 내 업무는 내가 더 잘 알지”라는 방어를 동시에 가집니다. 그래서 설명은 기능 이름보다 익숙한 업무 장면에서 시작해야 합니다. 챕터 5. 데이터 병아리반 — 숫자를 믿기 전에 질문하기라는 주제를 말하기 전에, 수강생이 어제 실제로 했을 법한 일 하나를 떠올리게 하세요.

강사는 빠르게 정답을 보여주기보다, 일부러 작은 시행착오를 남겨두는 편이 좋습니다. 첫 프롬프트는 조금 부족하게 넣고, 결과를 함께 보며 “무엇을 더 알려주면 좋아질까요?”라고 묻습니다. 이 순간 수강생은 Copilot을 마법 상자가 아니라 대화하며 다듬는 업무 파트너로 이해하기 시작합니다.

강사 “이번 장은 외우는 시간이 아닙니다. 여러분이 이미 하고 있는 업무를 Copilot에게 어떻게 설명하면 좋을지 같이 연습하는 시간입니다.”
수강생 예상 “제가 정확히 뭘 입력해야 할지 모르겠어요.”
강사 답변 “괜찮습니다. 처음부터 좋은 프롬프트를 쓰는 사람은 거의 없습니다. 먼저 거칠게 부탁하고, 나온 답을 보며 목적·대상·형식·주의사항을 하나씩 보태면 됩니다.”

수업 중 바로 쓸 확장 스크립트

“지금 Copilot이 준 답을 정답처럼 보지 말고, 신입 동료가 가져온 초안처럼 봅시다. 잘한 부분은 살리고, 빠진 배경은 추가하고, 위험한 표현은 고치면 됩니다. 오늘 우리가 배우는 것은 AI에게 일을 던지는 법이 아니라, AI와 함께 업무 초안을 다듬는 법입니다.”
