Streamlit Cloud 배포

오늘은 이런 일을 해요

이번 차시부터 프로젝트가 교실 안 파일에서 벗어나 웹으로 확장됩니다. Streamlit Cloud 배포에서는 데이터를 공유하고, 링크로 결과를 보여주는 방법을 배웁니다.

처음부터 자동화나 복잡한 개발을 하지 않습니다. 웹 화면에서 파일을 올리고, 템플릿을 수정하고, 공유 링크를 확인하는 방식으로 시작합니다.

목표는 개발자가 되는 것이 아니라, 내 프로젝트를 친구들이 실제로 열어 보고 이해할 수 있게 만드는 것입니다.

배포 링크배포 · 공유 · URLStreamlit 링크

오늘 한눈에 보기

따라하기

1. 파일 준비하기

CSV 파일의 열 이름과 단위가 맞는지 확인합니다.

2. 웹에 올리기

Google Sheets나 GitHub 웹 화면에서 파일을 업로드합니다. CLI는 사용하지 않습니다.

3. 링크 확인하기

공유 링크가 열리는지 친구 자리에서 확인합니다.

4. 여러 팀 데이터 합치기

공통 열 이름으로 팀별 데이터를 모아 비교합니다.

5. 기록 정리하기

README나 활동 기록지에 데이터 출처, 수정한 점, 한계를 적습니다.

실습 데이터

아래 내용은 오늘의 출발점입니다. 그대로 실행하거나 복사한 뒤, 표시된 부분을 우리 팀 상황에 맞게 바꿔 보세요.

team,timestamp,place,temp,humidity,light,sound,soil,note
A,2026-05-15 09:00,window,24.8,48,612,42,,맑음

# 숫자 칸에는 단위를 붙이지 않습니다. 예: 24.8℃가 아니라 24.8

AI와 함께 개선하기

AI는 여기서 처음부터 대신 만드는 도구가 아니라, 여러분이 만든 코드와 데이터를 더 읽기 쉽고 보기 좋게 다듬는 도구입니다. 작은 단위로 요청하고, 결과를 반드시 실행해 확인하세요.

내 CSV에는 team, timestamp, place, temp, humidity, light, sound 열이 있어. 이 데이터를 사용해서 현재 상태를 보여주는 Streamlit 지표 카드와 그래프를 더 보기 좋게 개선해줘. 바뀐 코드에는 주석을 달아줘.

점검하기

  • 오늘 만든 것이 실제로 작동하거나 화면에 보이나요?
  • 내가 바꾼 부분을 한 문장으로 설명할 수 있나요?
  • 친구 팀과 결과가 다르면 그 이유를 말할 수 있나요?
  • 데이터를 다뤘다면 시간, 장소, 조건이 함께 기록되어 있나요?
  • AI를 사용했다면 답을 그대로 붙이지 않고 검증했나요?
