Google Sheets 데이터 공책

오늘은 이런 일을 해요

이번 차시의 목표는 프로젝트가 내 컴퓨터 안에만 머물지 않게 하는 것입니다. Google Sheets 데이터 공책을 통해 데이터와 결과를 링크로 공유할 수 있게 만듭니다.

GitHub는 웹 화면으로 사용합니다. CLI 명령어를 외우는 시간이 아니라, 파일을 올리고 README를 고치고 링크를 확인하는 경험이 중요합니다.

공유는 편리하지만 책임도 생깁니다. 개인정보, 권한, 출처, 데이터 수정 기록을 함께 확인합니다.

오늘 흐름 그림

그림 24-1오늘 작업 흐름

장치를 새로 연결하기보다, 지금까지 만든 데이터와 기록을 화면·링크·발표로 정리합니다.

따라하기

1. 파일 준비하기

CSV 파일의 열 이름과 단위가 맞는지 확인합니다.

2. 웹에 올리기

Google Sheets나 GitHub 웹 화면에서 파일을 업로드합니다. CLI는 사용하지 않습니다.

3. 링크 확인하기

공유 링크가 열리는지 친구 자리에서 확인합니다.

4. 여러 팀 데이터 합치기

공통 열 이름으로 팀별 데이터를 모아 비교합니다.

5. 기록 정리하기

README나 활동 기록지에 데이터 출처, 수정한 점, 한계를 적습니다.

아래 내용은 오늘의 출발점입니다. 그대로 따라 한 뒤, 우리 팀 상황에 맞게 표시된 부분을 바꿔 보세요.

Google Sheets 업로드 전 확인

1. CSV 첫 줄에 열 이름이 있는가?
2. 숫자 칸에 ℃, dB 같은 단위를 붙이지 않았는가?
3. team, timestamp, place 열이 있는가?
4. 공유 권한은 수업에 필요한 범위로만 열었는가?
5. 링크를 친구 자리에서 열어 보았는가?

AI와 함께 개선하기

AI는 처음부터 대신 만드는 도구가 아니라, 여러분이 만든 코드와 데이터를 더 읽기 쉽고 보기 좋게 다듬는 도구입니다. 작은 단위로 요청하고, 결과를 반드시 실행해 확인하세요.

내 CSV에는 team, timestamp, place, light 열이 있어. 이 데이터를 사용해서 장소별 평균 밝기 그래프를 더 보기 좋게 개선해줘. 없는 열 이름은 만들지 말고, 바뀐 코드에는 주석을 달아줘.

점검하기

  • 오늘 만든 것이 실제로 작동하거나 화면에 보이나요?
  • 내가 바꾼 부분을 한 문장으로 설명할 수 있나요?
  • 친구 팀과 결과가 다르면 그 이유를 말할 수 있나요?
  • 데이터 출처와 한계를 적었나요?
  • AI를 사용했다면 답을 그대로 붙이지 않고 검증했나요?
