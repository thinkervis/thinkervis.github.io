챕터 5. 데이터 병아리반 — 숫자를 믿기 전에 질문하기

데이터 실습은 재미있지만 위험합니다. Copilot이 숫자를 말하면 그럴듯해 보이기 때문입니다. 데이터 수업의 핵심은 ‘분석해줘’가 아니라 ‘어떤 기준으로 분석했는지 설명해줘’입니다.

먼저 표를 이해했는지 확인합니다

수강생에게 CSV나 Excel을 주고 바로 인사이트를 달라고 하면 안 됩니다. 먼저 Copilot이 컬럼을 어떻게 이해했는지 확인해야 합니다. 컬럼 뜻을 잘못 이해하면 뒤의 분석도 모두 흔들립니다.

첨부한 CSV의 컬럼 구조를 먼저 설명해줘.
각 컬럼이 어떤 의미로 보이는지 추정하고, 분석 전에 확인해야 할 점을 알려줘.
그 다음 문의 유형 TOP 5, 평균 처리 시간이 긴 제품군, 재문의율이 높은 유형을 계산 기준과 함께 정리해줘.
잠깐만요 │ 계산 결과는 반드시 샘플로 검산합니다

상위 3개, 합계, 평균처럼 간단한 값은 사람이 직접 한두 개 확인해보는 습관을 들여야 합니다. Copilot이 계산 기준을 설명하지 못하면 결과를 그대로 믿으면 안 됩니다.

강사용 데이터 실습 운영법

좋은 데이터 실습은 작은 데이터로 시작합니다. 처음부터 복잡한 피벗이나 거대한 파일을 다루면 수강생은 Copilot보다 파일 처리에 지칩니다. 20~50행 정도의 샘플 데이터로 문의 유형, 처리 시간, 만족도처럼 이해하기 쉬운 컬럼을 사용하세요.

강사 “데이터 분석에서 첫 질문은 인사이트를 달라는 것이 아닙니다. 먼저 Copilot이 데이터를 어떻게 이해했는지 확인해야 합니다.”
수강생 예상 “결과가 맞는지 모르겠습니다.”
강사 답변 “좋은 의심입니다. 계산 기준을 설명하게 하고, 간단한 숫자는 직접 확인해봅시다. 데이터 분석에서 의심은 나쁜 태도가 아니라 좋은 태도입니다.”
