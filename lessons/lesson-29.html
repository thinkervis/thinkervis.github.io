오늘 만들 것

프로젝트 기획서

왜 배우나요?

마지막 단계는 지금까지 배운 모든 것을 하나의 팀 프로젝트로 묶는 시간입니다. 장치가 작동하고, 데이터가 쌓이고, 대시보드가 보여주고, GitHub가 기록해야 합니다.

프로젝트는 재미와 의미를 모두 가져야 합니다. 친구가 만져보고 반응을 볼 수 있으면서도, 데이터로 문제를 설명할 수 있어야 합니다.

완성보다 중요한 것은 다음 버전을 말할 수 있는 성장입니다. 한계와 개선 방향을 숨기지 않고 발표합니다.

준비물

  • Raspberry Pi Pico 2 W + Grove Shield
  • 오늘 활동에 맞는 Grove 센서 또는 출력 모듈
  • USB 케이블, 노트북, 팀 기록지
  • Google Sheets / GitHub / Streamlit 계정 또는 교사 제공 템플릿

핵심 개념

문제 정의 · 역할

따라하기

  1. 팀 역할을 정하고 오늘 완성할 최소 기능을 고릅니다.
  2. 장치, 데이터, 대시보드, GitHub 기록 중 부족한 부분을 먼저 보완합니다.
  3. AI는 코드 설명, 오류 점검, 발표 문장 다듬기에 사용하고 반드시 기록합니다.
  4. 친구 피드백을 받아 사용자가 이해하기 어려운 부분을 고칩니다.
  5. 최종 발표에서는 문제, 데이터, 작품, 한계, 다음 버전을 순서대로 말합니다.

실습 코드/데이터

# 프로젝트 본문 활동에서는 팀 코드와 데이터를 여기에 붙여 넣고 수정합니다.
# 1. 오늘 필요한 최소 기능을 먼저 실행합니다.
# 2. 기준값, 열 이름, 메시지 중 하나를 바꿉니다.
# 3. 실행 결과를 친구와 비교합니다.

장비와 라이브러리에 따라 함수명이나 핀 번호는 달라질 수 있습니다. 중요한 것은 구조를 읽고 우리 팀 상황에 맞게 고치는 것입니다.

바꿔보기 미션

  • 기준값, 색상, 메시지, 그래프 제목 중 하나를 우리 팀 상황에 맞게 바꿉니다.
  • 바꾼 전/후 결과를 사진, 표, 문장 중 하나로 기록합니다.
  • 친구 팀과 결과가 다르면 데이터나 기준값이 어떻게 다른지 설명합니다.
