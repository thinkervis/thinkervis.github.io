04차시 · 재미있는 손코딩

교실 소음 신호등

소리 크기에 따라 초록·노랑·빨강으로 바뀌는 신호등을 만든다.

오늘의 흐름

입력→조건/처리→색·소리·화면

오늘 만들 것

소음 신호등

왜 배우나요?

오늘 수업의 핵심은 '소음 신호등'이 실제로 눈앞에서 반응하는 경험입니다. 처음부터 모든 코드를 이해하려고 하기보다, 어느 줄을 바꾸면 결과가 달라지는지 관찰합니다.

팀원은 연결 담당, 코드 실행 담당, 기록 담당, 설명 담당으로 나눕니다. 역할은 다음 차시마다 바꿔서 모두가 장치와 코드에 손을 대게 합니다.

성공 기준은 화려한 작품이 아니라, 내가 바꾼 부분과 달라진 결과를 한 문장으로 설명하는 것입니다.

준비물

  • Raspberry Pi Pico 2 W + Grove Shield
  • 오늘 활동에 맞는 Grove 센서 또는 출력 모듈
  • USB 케이블, 노트북, 팀 기록지
  • CSV 양식 또는 활동 기록지

핵심 개념

센서값 흔들림 · 기준값 · 규칙

따라하기

  1. 완성 예시를 30초 동안 먼저 보고 오늘 만들 결과를 상상합니다.
  2. 피코와 모듈을 연결하고 기본 코드가 실행되는지 확인합니다.
  3. 교재에서 표시한 숫자, 기준값, 메시지 중 하나만 먼저 바꿔봅니다.
  4. 결과가 달라지면 왜 달라졌는지 팀 기록지에 적습니다.
  5. 친구 팀과 결과를 비교하고 우리 팀만의 색/규칙/설명을 붙입니다.

실습 코드/데이터

# 센서값을 읽고 기준에 따라 상태를 나누는 기본 구조입니다.
from time import sleep

LOW = 300
HIGH = 700

while True:
    value = read_sensor()  # 예: 조도, 소리, 온습도 센서값
    print("sensor:", value)

    if value < LOW:
        state = "낮음"
    elif value < HIGH:
        state = "보통"
    else:
        state = "높음"

    print("state:", state)
    sleep(1)

# 바꿔보기: LOW와 HIGH 기준값을 우리 팀 데이터에 맞게 바꿔보세요.

장비와 라이브러리에 따라 함수명이나 핀 번호는 달라질 수 있습니다. 중요한 것은 구조를 읽고 우리 팀 상황에 맞게 고치는 것입니다.

바꿔보기 미션

  • 기준값, 색상, 메시지, 그래프 제목 중 하나를 우리 팀 상황에 맞게 바꿉니다.
  • 바꾼 전/후 결과를 사진, 표, 문장 중 하나로 기록합니다.
  • 친구 팀과 결과가 다르면 데이터나 기준값이 어떻게 다른지 설명합니다.
