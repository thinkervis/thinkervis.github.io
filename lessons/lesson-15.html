오늘 만들 것

왜 배우나요?

AI를 쓰기 전에 먼저 AI를 의심하는 법을 배웁니다. AI는 빠르지만, 우리 교실의 측정 상황과 데이터 맥락을 자동으로 알지는 못합니다.

AI가 준 코드는 실행 전에 읽고, 실행 후에는 결과가 우리 데이터와 맞는지 확인합니다.

AI를 사용한 부분은 숨기지 않고 질문, 답변, 수정, 검증 방법을 기록합니다.

준비물

  • Raspberry Pi Pico 2 W + Grove Shield
  • 오늘 활동에 맞는 Grove 센서 또는 출력 모듈
  • USB 케이블, 노트북, 팀 기록지
  • CSV 양식 또는 활동 기록지

핵심 개념

프롬프트 · 데이터 구조

따라하기

  1. AI가 만든 설명이나 코드를 먼저 읽고 이상한 부분을 표시합니다.
  2. 우리 데이터 열 이름, 단위, 목적을 프롬프트에 정확히 넣습니다.
  3. AI 답변을 실행하기 전에 예상 결과를 적습니다.
  4. 실행 후 실제 결과와 비교하고 틀린 부분을 수정합니다.
  5. AI 활용 기록지에 질문, 사용한 부분, 수정한 부분, 검증 방법을 남깁니다.

- 열 이름이 실제 데이터와 같은가?
- 그래프 축과 단위가 맞는가?

장비와 라이브러리에 따라 함수명이나 핀 번호는 달라질 수 있습니다. 중요한 것은 구조를 읽고 우리 팀 상황에 맞게 고치는 것입니다.

바꿔보기 미션

  • 기준값, 색상, 메시지, 그래프 제목 중 하나를 우리 팀 상황에 맞게 바꿉니다.
  • 바꾼 전/후 결과를 사진, 표, 문장 중 하나로 기록합니다.
  • 친구 팀과 결과가 다르면 데이터나 기준값이 어떻게 다른지 설명합니다.
