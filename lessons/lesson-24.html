오늘 만들 것

클라우드 데이터 시트

왜 배우나요?

프로젝트가 개인 파일에서 벗어나 클라우드와 웹으로 확장됩니다. Google Sheets는 공동 데이터 공책, GitHub는 프로젝트 전시 폴더, Streamlit Cloud는 대시보드 전광판입니다.

처음에는 자동화보다 수동 업로드와 템플릿 수정으로 시작합니다. 실패 없이 흐름을 이해하는 것이 먼저입니다.

API는 선택 확장입니다. 외부 데이터가 우리 센서 해석을 더 풍성하게 만들 때만 붙입니다.

준비물

  • Raspberry Pi Pico 2 W + Grove Shield
  • 오늘 활동에 맞는 Grove 센서 또는 출력 모듈
  • USB 케이블, 노트북, 팀 기록지
  • Google Sheets / GitHub / Streamlit 계정 또는 교사 제공 템플릿

핵심 개념

공유 · 표준 열 · 품질

따라하기

  1. 팀 데이터 파일과 공통 열 이름을 확인합니다.
  2. Google Sheets 또는 GitHub 웹 화면에서 파일을 업로드하거나 수정합니다.
  3. Streamlit Cloud 또는 샘플 JSON을 이용해 웹에서 볼 수 있는 결과를 만듭니다.
  4. 여러 팀 데이터나 외부 데이터와 비교할 때 해석의 한계를 적습니다.
  5. 공유 링크와 기록 파일을 README에 정리합니다.

실습 코드/데이터

team,timestamp,place,temp,humidity,light,sound,soil,note

A,2026-05-15 09:05,back,25.1,50,388,45,,조금 어두움
B,2026-05-15 09:10,hall,23.9,53,455,58,,쉬는 시간

# 주의: 숫자 칸에는 단위를 붙이지 않습니다. 예: 24.8℃가 아니라 24.8

장비와 라이브러리에 따라 함수명이나 핀 번호는 달라질 수 있습니다. 중요한 것은 구조를 읽고 우리 팀 상황에 맞게 고치는 것입니다.

바꿔보기 미션

  • 기준값, 색상, 메시지, 그래프 제목 중 하나를 우리 팀 상황에 맞게 바꿉니다.
  • 바꾼 전/후 결과를 사진, 표, 문장 중 하나로 기록합니다.
  • 친구 팀과 결과가 다르면 데이터나 기준값이 어떻게 다른지 설명합니다.
